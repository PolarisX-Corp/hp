記事本文で使える Markdown をひと通り並べた見本です。実際の記事と同じ `.bl-prose` で描画しているため、ここでの見え方がそのまま記事の見え方になります。**このページは開発時のみ表示され、本番ビルドでは 404 になります。**

書き方のルールは [`public/blog/README.md`](https://github.com/PolarisX-Corp/hp/blob/main/public/blog/README.md) を参照してください。

> **見出しレベルの原則**
>
> - 記事タイトルは frontmatter の `title` が H1 になります。本文で `# ` は使いません。
> - 本文の最上位は `## `（H2）。以降は `### `（H3）→ `#### `（H4）と 1 段ずつ下げます。
> - 見た目を変えたいという理由でレベルを飛ばさないでください。目次とスクリーンリーダーの構造が壊れます。

## H2 — 章の見出し

H2 は記事の章にあたる、いちばん強い区切りです。塗りと太い左バーで、スクロール中でも「ここから話題が変わる」ことが分かるようにしています。`## ` は自動で目次に並びます。

段落はそのまま書きます。**太字**、_斜体_、`インラインコード`、[リンク](https://polarisx.co.jp/)、そして ==マーカー== が使えます。マーカーは結論やキーワードなど、流し読みでも拾ってほしい短いフレーズに、1 段落 1 箇所を目安に使います。

### H3 — 節の見出し

H3 は章の中の区切りです。左バーと下線だけで、H2 より一段弱い表現にしています。H3 も目次に並びます。

#### H4 — 項の見出し

H4 は H3 の中をさらに分けたいときだけ使います。タイポグラフィのみで区別し、目次には出ません。ここまで細かく割る必要があるなら、章立て自体を見直したほうが読みやすくなることが多いです。

## リスト

### 箇条書き（ul）— 並列に並ぶ要素

順序に意味がないものは箇条書きにします。

- 短く言い切る。1 項目 1 メッセージにする
- 文末の句点は、文になっている場合だけ付ける
- 入れ子は 1 段まで
  - 入れ子の点は白抜きになり、親と区別できます
  - 2 段以上は読みにくいので、見出しで割るか表にします

### 番号付きリスト（ol）— 手順・順位

順序や件数に意味があるものは番号付きにします。

1. 目的と読者を決める
2. 見出し（H2）だけで骨子を作る
3. 各見出しの下に段落を埋める
4. 最後に要点カードとマーカーを足す

### チェックリスト

- [x] frontmatter を埋めた
- [x] 画像を `assets/` に置いた
- [ ] 320px 幅で表示崩れがないか確認した

## 引用 — 要点カード・注記

`>` の引用は、本文から浮かせたい「要点」「注記」「補足」に使います。1 行目を太字にすると見出し行として機能します。

> **この記事の要点**
>
> - 結論を先に 3 点ほど箇条書きで示すと読了率が上がります
> - 各見出し（`## `）は自動で目次に並びます
> - 引用は装飾ではなく、本文と役割が違うことを示すために使います

> **注意**：個人情報や機密情報を含む操作は、必ず人の確認を挟んでください。

出典を示す本来の引用としても同じ見た目になります。用途を分けたい場合は、1 行目のラベル（**要点** / **注意** / **補足** など）で書き分けてください。

## 表

列が多い表や長い文字列を含む表は、狭い画面では表だけが横スクロールします。本文が押し出されたり見切れたりしません。

| 記法 | 用途 | 目次に載る |
| --- | --- | --- |
| `## ` | 章の見出し（H2） | 載る |
| `### ` | 節の見出し（H3） | 載る |
| `#### ` | 項の見出し（H4） | 載らない |
| `- ` | 並列の箇条書き | — |
| `1. ` | 手順・順位 | — |
| `> ` | 要点カード・注記 | — |
| `==テキスト==` | マーカー強調 | — |

## コード

インラインは `npm run dev` のように書きます。ブロックは言語を指定します。

```bash
git checkout -b blog/your-article-slug
git add public/blog/your-article-slug
git commit -m "blog: <記事タイトル>"
```

```json
{
  "title": "記事タイトル",
  "category": "ai-agent",
  "publishedAt": "2026-01-01"
}
```

## 区切り線

話題が大きく変わるが章を分けるほどではない、というときだけ使います。多用しないでください。
